### **Phase 4: Rate Limiter Feature Demo (2 minutes)**

**Objective**: Show rate limiting in action or explain behavior

**Steps**:

**Option A (If demo environment available)**:
1. Show Rate Limiter configuration in code/environment:
   ```
   RATE_LIMIT_THRESHOLD=10
   RATE_LIMIT_WINDOW=900 (15 seconds)
   ```
2. Make rapid consecutive requests to POST /expenses:
   - Requests 1-10: 201 Created (success)
   - Request 11: 429 Too Many Requests (rate limited)
3. Explain: "Rate limiter protects API from abuse; clients see 429 and retry"

**Option B (If no demo environment)**:
1. Show Rate Limiter middleware code:
   ```javascript
   // Rate limiter: per-IP tracking, sliding window
   const isRateLimited = (ip) => {
     const count = getRateLimitCount(ip);
     return count >= THRESHOLD;
   };
   ```
2. Explain: "Middleware intercepts every request, checks rate; rejects if threshold exceeded"
3. Show test case demonstrating 429 response
4. Explain: "Production will monitor rate-limit headers; allows graceful client backoff"

**Expected Outcome**: Mentor understands rate limiting behavior and purpose

**Fallback**: If time is tight (4+ min already spent), summarize: "Rate limiter configured with 10 req/15sec threshold; all tests passing; ready for production"

**Timing Note**: Can flex to 1.5 or 2 minutes depending on prior phases; optional demo can be skipped
